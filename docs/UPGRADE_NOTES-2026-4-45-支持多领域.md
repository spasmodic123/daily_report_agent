# 多领域日报Agent改进说明

## 改进概述

已成功将单一AI领域的日报agent扩展为支持多领域（AI + 财经 + 医疗健康）的综合资讯系统。

## 主要改动

### 1. 新增文件

#### `domain_config.py` - 领域配置中心
- 定义了三个领域的完整配置：
  - **AI行业**：8个官方站点 + 7个媒体站点
  - **财经**：8个官方站点 + 9个媒体站点  
  - **医疗健康**：10个官方站点 + 8个媒体站点
- 每个领域包含：
  - 搜索查询词（中英文）
  - 域名白名单（官方源 + 媒体源）
  - 定制化提炼提示词模板
  - 关注重点说明
- 提供统一报告排版提示词模板

### 2. 修改文件

#### `source_filter.py` - 扩展白名单
新增权威媒体源（共60个域名）：

**财经领域：**
- 官方机构：IMF, World Bank, Federal Reserve, ECB, 中国人民银行, 证监会, 上交所, 深交所
- 媒体：财新网, 第一财经, 财联社, 东方财富, 新浪财经

**医疗健康领域：**
- 官方机构：WHO, NIH, FDA, CDC, Nature, NEJM, The Lancet, 国家卫健委, 国家药监局, 中国疾控中心
- 媒体：STAT News, MedPage Today, 丁香园, 健康界, 医学界, 生物谷

#### `index.py` - 重构主流程
**核心改动：**

1. **新增 `call_tavily_search_multi_domain()`**
   - 替代原有的 `call_tavily_search()`
   - 支持多领域并发搜索
   - 返回按领域分组的搜索结果

2. **重构 `handler()` 函数**
   - Stage 1: 多领域并发搜索
   - Stage 2-4: 按领域循环处理（提取、清洗、提炼、过滤）
   - Stage 5: 统一报告排版与推送

**处理流程：**
```
启动 → 获取启用领域 → 多领域并发搜索
  ↓
对每个领域：
  → Tavily提取全文 → LLM清洗噪声 → 领域定制提炼 → 域名过滤
  ↓
统一报告排版 → 推送钉钉
```

### 3. 测试文件

#### `test_multi_domain.py`
- 测试领域配置加载
- 验证查询词、域名白名单数量

#### `test_whitelist_coverage.py`
- 测试白名单覆盖率
- 确保所有配置的站点都在过滤器白名单中

## 技术亮点

### 1. 配置驱动架构
- 领域配置与业务逻辑分离
- 新增领域只需修改 `domain_config.py`
- 无需改动主流程代码

### 2. 领域定制化
- 每个领域有独立的搜索查询词
- 每个领域有定制的提炼提示词
- 关注重点不同（AI关注技术参数，财经关注数据指标，医疗关注临床数据）

### 3. 并发优化
- 多领域搜索并发执行
- 页面清洗并发执行
- 提升整体处理速度

## 使用方式

### 运行日报生成
```bash
python index.py
```

### 测试配置
```bash
# 测试领域配置
python test_multi_domain.py

# 测试白名单覆盖率
python test_whitelist_coverage.py
```

### 自定义领域
如需新增领域（如"科技综合"、"政策法规"），只需：

1. 在 `domain_config.py` 的 `DOMAINS` 字典中添加新领域配置
2. 在 `source_filter.py` 中添加该领域的权威媒体源
3. 在 `UNIFIED_REPORT_PROMPT_TEMPLATE` 中添加对应的 `{new_domain}_facts` 占位符

无需修改主流程代码。

## 注意事项

1. **API调用成本**：多领域会增加Tavily搜索和LLM调用次数，请注意API配额
2. **处理时间**：多领域并发处理，但总体时间会比单领域长
3. **钉钉消息长度**：统一报告可能较长，注意钉钉消息长度限制
4. **领域平衡**：如某领域当日无重要资讯，会显示"今日暂无重大动态"
